# Forge v2 Epic #180 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the Forge v1 public contract with the 28-tool Forge v2 contract, backed by typed schemas, journaled mutation, impact-routed verification, release-gate corpora, lean payloads, and a reversible big-bang cutover.

**Architecture:** Preserve proven repository, Git, GitHub, policy, state, and shipping use cases behind new composite application services. Add a Pydantic contract registry that is authoritative for discovery and runtime validation, a journaled filesystem transaction port/adapter for all workspace mutations, and a provider-neutral code-intelligence port used by verification routing. The v2 MCP server exposes exactly 28 static tools and a separate grace server exposes one migration error for the retired identity.

**Tech Stack:** Python 3.10+, Pydantic 2, FastMCP/MCP 1.28.1, Git CLI, pytest, Ruff, Mypy, JSON golden contracts, deterministic local corpora.

## Global Constraints

- Exactly 28 public v2 tools; no hidden stage-specific tools.
- `workspace_mutate` is the only public filesystem mutation tool.
- No one-way migration of workspace, audit, operation, or configuration-generation state.
- Public results and audit evidence contain no absolute host paths.
- Outputs use closed schemas; clients follow tolerant-reader semantics for additive fields.
- Low-confidence impact evidence always falls back to the full verification profile.
- Commit and push gates remain bound to the exact verified workspace fingerprint.
- No arbitrary shell, force push, protected-branch write, secret persistence, or direct merge capability.
